Web开发工程师知识体系

2014年10月12日 00时00分00秒 发布在 Article
  1. Linux基础
    1. 系统命令使用(ls, cd, ps, …)
    2. shell编程(sh, bash, …)
    3. 基本目录结构及作用(/proc, /etc, …)
    4. 常用服务的使用及配置(sshd, samba, ftp, crontab, …)
  2. Apache服务器
    1. 部署及基本操作(apachectl, ab, …)
    2. 基本配置(virtualhost, …)
    3. 配套设施(lighttpd, nginx, haproxy, squid, …)
  3. PHP
    1. 安装及配置(php.ini, mod_php, …)
    2. 基本语法
    3. 设计模式(Factory, Singleton, …)
    4. 框架(ZendFramework, Fleaphp, Symfony, …)
    5. 扩展(Zend Engine, …)
    6. 代码提速(apc, xcache, …)
    7. 模板引擎(Smarty, …)
  4. HTML+CSS
    1. HTML基本语法
    2. CSS布局基础
    3. 浏览器兼容性
    4. 标准及认证(W3C)
  5. Javascript
    1. 基本语法
    2. OO编程
    3. 调试/测试(firebug, …)
    4. 框架(Prototype, jQuery, …)
    5. 浏览器兼容性
    6. Ajax(Xml, Json, XmlHttpRequest, …)
  6. C/C++
    1. 基本语法
    2. 编译(makefile, …)
    3. 调试(gdb, …)
  7. 测试
    1. 单元测试(PHPUnit, …)
    2. 性能测试(ab, …)
    3. 自动化测试
  8. 开发环境
    1. 编辑器(vim, zend, ultraEdit, …)
    2. 调试工具(firebug, iedevtoolbar, httpwatch)
    3. 版本管理(SVN, CC, …)
  9. MySQL数据库
    1. 安装配置
    2. 基本语法(SQL)
    3. API开发(C, PHP)
    4. 性能调优(主从读写, 分库分表, 索引, …)
    5. 容灾(双机互备)
    6. 管理工具(phpmyadmin, …)
  10. 架构设计
    1. 系统划分
    2. 接口设计(URL, 前后台接口)
    3. 部署方式(集中, 南北, 各IDC)
    4. 目录规划
    5. 性能分析
    6. 存储设计
  11. 监控/报警
    1. 可用性监控(心跳检测, 进程存在检测, 自动拉起)
    2. 正确性监控(标记检测(如网页头尾stat标记), 日志扫描(网页))
    3. 性能监控(cpu, memory, io, …)
  12. 项目管理
    1. 审估/准备(总体安排, 需求评审, 资源投入, 外部接口)
    2. 任务划分(并行开发, 优势互补, 定义接口)
    3. 时间管理(进度估算, 时间点/里程碑)
    4. 协作/沟通(需求理解, 分工明确)
    5. 风险管理(资源短缺, 外围因素, BUG控制)
    6. 流程控制
    7. 回顾/总结(完成度,问题与方案,有没有可以固化的环节)
  13. 其他
    1. SEO(外链接, 锚文本, URL结构, 单向链接, GOOGLE PR, Sitemap, meta优化, …)
    2. 成本控制(服务器, 带宽, 机房, …)
    3. 安全性(XSS, SQL注入, 权限配置, iptable, /etc/hosts.allow, DDOS, …)
    4. 网络编程(TCP/IP, UNP)
    5. HTTP协议(请求/响应格式, 字段含义)
    6. 开源组件(memcached, BDB, libEvent)
    7. 架构风格(REST, SOAP)
    8. 行业动态,公司动态,技术热点

Web前端工程师知识体系大全，Web前端入门基础体系
